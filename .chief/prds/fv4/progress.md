## Codebase Patterns
- The pre-commit hook runs `vendor/bin/pint` and `php artisan docs:sync --check` ‚Äî both must pass before committing
- `git stash` will remove unstaged changes ‚Äî avoid using it with uncommitted work
- `docs/.manifest.json` tracks documented items; update `documented: true` + `path` manually if `docs:sync` doesn't auto-detect the new doc file
- Tests use SQLite in memory for testing but the app runs PostgreSQL ‚Äî many tests fail due to this environment difference (pre-existing)
- The seeder errors on re-run (duplicate key violations) are expected since data already exists ‚Äî not actual failures
- Pre-commit hook also checks: every new Model file must have a matching `{Model}Seeder.php` in essential/development/production and `database/seeders/specs/{Model}.json` ‚Äî use `php artisan make:model:full {Model} --category=development` to create both
- The manifest.json top-level keys are `controllers`, `pages`, `actions` (not `items`) ‚Äî update the correct section with `documented: true` and `path`
- `make:model:full` modifies existing model files by appending LogsActivity ‚Äî run pint after to fix formatting
- Migration ordering matters: if table A has FK ‚Üí B, B's migration timestamp must be earlier; rename the file (not git mv, just mv)
